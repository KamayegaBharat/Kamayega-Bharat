# Contributing to Kamayega Bharat 🚀

First of all, thank you for considering contributing to Kamayega Bharat! We are building the fastest, most efficient workforce platform for India, and we are glad you're here.

## ⚡ Our Core Mission: The 100/100 Standard
We obsess over performance. Every contribution is evaluated against our 100/100 Google PageSpeed benchmark. If a change slows down the site, we iterate until it doesn't.

### 🛡️ Performance Guardrails:
* No Heavy Libraries: Do not add new NPM packages without a deep discussion. We prefer lightweight, native solutions.
* Asset Optimization: All images must be in WebP/SVG format and compressed.
* Bundle Size: We prioritize "Tree-shaking." Only import the specific Firebase modules you need.
* Mobile First: 90% of our users are on mobile. High-performance on low-end devices is our priority.

---

## 🛠️ How to Contribute

### 1. Reporting Bugs 🐛
If you find a bug, please open an Issue and include:
* A clear description of the bug.
* Steps to reproduce the issue.
* Your device specs (e.g., Android/iOS, Chrome/Safari).
* A screenshot of the Lighthouse/PageSpeed score if the bug is performance-related.

### 2. Suggesting Features ✨
We love new ideas! However, please open an Issue to discuss the feature before writing any code. We need to ensure every new feature aligns with our "Speed First" philosophy.

### 3. Pull Requests (PRs) 📥
1. Fork the repository and create your branch from main.
2. Ensure your code follows the existing project structure.
3. Test locally: Run a performance audit before submitting.
4. Describe your changes: In the PR description, explain *what* you changed and *how* it affects (or protects) the site's speed.
